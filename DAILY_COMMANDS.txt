============================================================
  MIROTRADE FRAMEWORK — DAILY STARTUP & OPERATIONS GUIDE

PREREQUISITES (one-time, already done)
---------------------------------------
- MetaTrader 5 desktop app must be OPEN and LOGGED IN before running anything
- .env file must exist at: D:\Trading Project\MiroTrade-Framework\.env
- Python environment must have all packages installed

------------------------------------------------------------
STEP 1 — OPEN MT5 FIRST
------------------------------------------------------------
Open MetaTrader 5 manually and log in to your broker account.
Make sure XAUUSD is visible in the market watch window.
Keep MT5 running in the background. Do NOT close it.

------------------------------------------------------------
STEP 2 — OPEN A TERMINAL (Command Prompt or PowerShell)
------------------------------------------------------------
Press Win+R → type cmd → Enter
OR open VS Code terminal

Navigate to the project:
  cd "D:\Trading Project\MiroTrade-Framework"

------------------------------------------------------------
STEP 3 — START THE FULL SYSTEM
------------------------------------------------------------
Run this ONE command. It starts all 31 agents automatically:

  python launch.py

You will see all agents launch one by one and get a startup
summary. Keep this terminal OPEN and RUNNING all day.

Expected startup output:
  Paper Trader     : STARTING
  News Sentinel    : STARTING (AI-powered Claude / Rule-based)
  Risk Manager     : STARTING
  Orchestrator     : STARTING
  ...all 31 threads...
  All 31 agents launched!
  Dashboard → http://localhost:5055

------------------------------------------------------------
STEP 4 — OPEN THE DASHBOARD
------------------------------------------------------------
Open your browser and go to:
  http://localhost:5055

This shows everything:
  - Paper account balance and performance
  - Live confluence ring (9 factors)
  - Orchestrator GO/NO-GO status
  - All agent health (green = active, red = offline)
  - Open trades, equity curve, recent trades
  - Market narrative, news brain, DXY/yields
  - Deployment readiness checklist

------------------------------------------------------------
STEP 5 — START THE TRADINGVIEW WEBHOOK (OPTIONAL)
------------------------------------------------------------
Only needed if you are using TradingView Pine Script alerts.
Open a SECOND terminal:

  cd "D:\Trading Project\MiroTrade-Framework"
  python tradingview/bridge_launcher.py

Webhook listens at: http://localhost:5056/webhook
Keep this terminal open separately.

------------------------------------------------------------
DAILY HEALTH CHECK (do this once after startup)
------------------------------------------------------------
Look at the dashboard System Status Bar at the top:

  NEWS  → should say OK or SAFE (not BLOCK)
  RISK  → should show a score like 8/10 or 9/10
  ORCH  → should say GO (means all systems green)
  MTF   → should say BUY or SELL (not NEUTRAL)
  MT5   → should say LIVE (means MT5 is connected)

If ORCH says NO-GO, check the Orchestrator section on the
dashboard to see which check is failing.

------------------------------------------------------------
IF SOMETHING CRASHES / SYSTEM RESTARTS
------------------------------------------------------------
Just run:  python launch.py   again from Step 3.
All state is saved to JSON files — nothing is lost on restart.
The system picks up where it left off.

WHAT RUNS AUTOMATICALLY (you don't need to do anything)
------------------------------------------------------------
  Every 60 seconds   → Orchestrator makes GO/NO-GO decision
  Every 60 seconds   → Paper Trader checks open trades for SL/TP
  Every 5 minutes    → H1 scan for new trade signals
  Every 60 seconds   → M5 scan for new trade signals
  Every 5 seconds    → Price feed updates live_price.json
  Every 30 minutes   → News Brain fetches headlines (NewsAPI)
  Every 15 seconds   → Scale Out checks for partial close triggers
  Every 15 seconds   → Breakeven Guard checks for SL moves
  Every 3 seconds    → Dashboard refreshes all data

  03:30 IST daily    → Morning briefing sent to Telegram
  16:30 IST daily    → Evening summary sent to Telegram
  18:30 IST daily    → Nightly optimizer runs (30 combos, ~5-10min)
                        Best params auto-applied if improvement ≥3% WR

------------------------------------------------------------
ENABLING LIVE TRADING (Option B — Python v15F → MT5)
------------------------------------------------------------
When you are ready to trade real money with the v15F signals:

1. Add this line to your .env file:
     LIVE_MODE=true

2. Restart:  python launch.py

That's it. The paper trader will now mirror every v15F paper signal
to MT5 via the bridge. Lot size is calculated from your real MT5
account balance (not the $10k paper balance).

IMPORTANT: Do NOT enable until ALL go-live conditions below are met.
When live, the paper account still tracks everything — you can compare
paper vs live performance side by side on the dashboard.
